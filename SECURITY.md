# Security Policy

## Supported Versions

Мы поддерживаем только актуальную версию в ветке `main` (последний деплой).
Все старые версии считаются неподдерживаемыми.

## Reporting a Vulnerability

Пожалуйста, **не создавайте публичные Issues** для уязвимостей.

1) Используйте **GitHub Private vulnerability reporting** (Security → “Report a vulnerability”), если доступно.  
2) Если нет — напишите приватно владельцу репозитория.

### Что приложить
- Краткое описание и влияние (что можно украсть/сломать)
- Шаги воспроизведения / PoC
- Скриншоты/логи (без секретов)
- Версия/коммит (если знаете)

### SLA
- Ответ: в течение **72 часов**
- Исправление критичных: как можно быстрее, обычно **до 7 дней**
